# GitHub Pages Deploy

Seat Happens now uses GitHub Actions for Pages deployment.

## One-time setup

1. Create or open the GitHub repository.
2. Add the remote locally:

   `git remote add origin <your-repo-url>`

3. Push `main`:

   `git push -u origin main`

4. In GitHub, open `Settings > Pages`.
5. Set `Source` to `GitHub Actions`.

The repository already contains:

- `.github/workflows/check-publish-sync.yml`
- `.github/workflows/deploy-pages.yml`

After the first push, GitHub Actions will deploy the contents of `public/`.

## Normal publish flow

1. Edit `study_group_supabase.html`
2. Run `./scripts/sync-pages.ps1`
3. Commit and push to `main`
4. Wait for the `Deploy GitHub Pages` workflow to finish

The sync check workflow fails if `public/index.html` is not aligned with `study_group_supabase.html`.

## Notes

- `public/index.html` stays in the repo for local preview and as the Pages artifact source.
- `public/.nojekyll` is generated by `scripts/sync-pages.ps1`.
- Supabase remains the login and database backend. Do not upload any secret keys.
